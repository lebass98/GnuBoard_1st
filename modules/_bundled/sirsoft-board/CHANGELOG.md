# Changelog

이 프로젝트의 모든 주요 변경사항을 기록합니다.
형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/)를 따르며,
[Semantic Versioning](https://semver.org/lang/ko/)을 준수합니다.

## [1.0.1] - 2026-07-08

### Added

- 관리자 게시물 조회 화면(목록·상세)에 "유저 화면 보기" 버튼을 추가했습니다. 클릭하면 해당 게시판/게시글의 실제 사용자 화면이 새 탭으로 열립니다.
- 사용자 게시판 화면(목록·상세)에 관리자 진입 버튼을 추가했습니다. 게시판 관리 권한이 있는 사용자에게만 "관리자 게시물 조회"·"게시판 관리" 버튼이 보이며, 클릭하면 관리자 화면이 새 탭으로 열립니다.

### Changed

- 관리자 게시판 설정 화면의 주소가 게시판 번호 대신 게시판 아이디(슬러그) 기준으로 바뀌었습니다 (예: /admin/boards/free/edit). 새 게시판을 만든 직후 이동하는 편집 화면, 목록의 활성화 토글·삭제, 목록에서 여는 수정 화면, 상품 문의 설정의 "관리자 메뉴에 추가"까지 모두 게시판 아이디 기준으로 통일되었습니다. 기존 번호 기반 주소도 계속 동작합니다.

## [1.0.0] - 2026-07-01

### Added

- 관리자 대시보드의 게시판 "오늘 새 글/새 댓글" 배지를 클릭하면 게시판 관리 화면으로 바로 이동합니다.
- 게시판 생성/수정 화면에 "관리자 메뉴에 표시" 토글을 추가했습니다. 켜고 저장하면 좌측 관리자 메뉴의 "게시판 관리" 아래에 해당 게시판이 표시되고, 끄고 저장하면 메뉴에서 제거됩니다. 이전에는 게시판을 만든 뒤 수정 화면에서만 메뉴에 추가할 수 있었습니다. 게시판을 복사할 때도 원본의 관리자 메뉴 표시 상태가 복사본에 그대로 이어집니다.

#### 레이아웃 편집기

- 게시판 화면을 샘플 데이터로 미리보고 편집할 수 있습니다 — 게시글 목록·상세·댓글, 게시판 관리/신고 화면이 샘플 데이터로 표시되며, 일반 글/비밀글 잠금, 글 있음/없음 같은 화면 상태도 미리보기로 전환해 볼 수 있습니다.
- 화면 상태 미리보기를 더 많은 게시판 화면으로 넓혔습니다 — 게시판 목록의 비어 있음, 글 작성/수정, 관리자의 게시판·게시글 폼(신규/수정/답글), 그리고 게시판 설정의 모든 탭(일반·기본값·신고 정책·스팸 보안·SEO·알림 정의·본인인증 정책)을 전환해 미리볼 수 있습니다.
- 데이터 소스 목록에서 이 확장이 제공하는 데이터 소스가 친화 명칭으로 표시되고, 어느 확장이 제공했는지 출처가 함께 표시됩니다.
- [검색엔진] 탭에서 게시글의 공유 이미지·제목·구조화 데이터 등 이 확장이 제공하는 SEO 자동값이 "게시글 제목"·"대표 이미지" 같은 데이터 연결 칩(출처 표시 포함)으로 보이고, 다른 데이터로 바꿔 지정할 수 있습니다.

#### 관리자 대시보드 게시판 영역

- 관리자 대시보드 상단에 게시판 / 신고 빠른 진입 버튼을 추가 — 게시판 모듈이 활성화된 환경에서만 노출되며, 비활성화 시 자동으로 사라짐
- 관리자 대시보드 중앙에 게시판 통계 카드 영역 추가 — 오늘의 새 글·새 댓글 배지, 7일 게시글·댓글 추세 그래프(직전 7일 대비 증감율 포함), 최신 게시글 5건, 미처리 신고 5건 카드로 구성. 라이트/다크 모드 양쪽에서 텍스트·구분선·배지가 일관되게 보이도록 색상 페어 적용
- 게시판 모듈을 비활성화하면 위 버튼·카드 영역이 모두 사라지고, 재활성화하면 새로고침으로 자동 복귀
- 매 시간 자동 실행되는 게시판 집계 스케줄러 추가 — 게시글·댓글 통계를 일별로 누적하여 대시보드 카드와 추세 그래프의 데이터 출처가 됨. 수동 실행도 가능

### Changed

- 코어 최소 요구 버전을 7.0.0-beta.8 로 상향. SEO 자동값 연결 칩 편집 기능이 코어 7.0.0-beta.8 에서 제공하는 편집기 연동 표면을 사용합니다.
- 게시글·댓글 작성일 표시 형식을 코어 표준(`Y-m-d H:i:s`)으로 통일 — 이전의 한글 요일 포함 표기가 영문/일본어 사이트에서도 한글로 출력되던 다국어 미정합과 이커머스 주문일시와의 표기 불일치를 해소. 시드/시계 어긋남 등으로 미래 시각이 들어오는 경계 케이스도 안정적으로 처리
- 신고 정책 설정 화면의 알림 영역을 개편 — 별도 활성화 토글을 제거하고, 신고 접수·처리 알림이 채널별(이메일/사이트 알림)로 어떤 템플릿으로 발송되는지와 활성/비활성 상태를 한눈에 보여줍니다. 채널·템플릿·수신자 관리는 "알림 설정으로 이동" 버튼으로 알림 설정 메뉴에서 일원화했습니다.
- 게시판 설정의 금지어·허용 확장자 입력 방식을 태그 입력으로 통일했습니다 — 쉼표로 구분해 한 줄에 적던 방식에서, 단어를 입력하고 Enter 를 누르면 칩으로 하나씩 추가되는 방식으로 바뀌어 입력·삭제가 직관적입니다.
- 관리자 게시글 상세의 댓글 카드 우측 버튼 영역(답변/신고/더보기) 사이 간격을 일관되게 조정 — 버튼이 붙어 있던 시각 결함 해소

#### 관리자 화면 외형 정합

- 게시판 관리 전반(환경설정 전 탭·게시판 폼·게시글 폼·신고 상세·목록·모달·검색바)의 카드·입력 칸·라벨·본문 텍스트·구분선 모양과 간격을 다른 관리자 화면과 동일한 결로 통일했습니다 — 화면마다 조금씩 다르게 보이던 카드 모서리·여백·헤더 간격, 입력 칸 테두리·배경, 글자 톤이 한 가지 기준으로 정합됩니다.
- 게시판 생성/수정 폼 "권한" 탭의 접근 제어 영역과 신고 상세 화면에서 항목·카드 사이 간격이 의도보다 부풀어 보이던 회귀를 바로잡아 다른 탭·화면과 동일한 간격으로 표시됩니다.
- 게시판 신고 정책 탭의 "채널별 발송 상태"에서 사이트 알림/이메일 아이콘이 라벨 위로 올라가 비대칭으로 보이던 회귀를 바로잡아, 아이콘이 채널 라벨 옆 가운데로 정렬됩니다.
- 이커머스 환경설정의 문의 게시판 선택 항목에서 라벨과 선택 박스 사이에 옅은 분리선·간격이 끼어 보이던 회귀를 바로잡아 한 필드로 응집됩니다.
- 게시판 댓글 패널 상단 헤더의 하단 분리선이 흐려져 헤더와 본문 경계가 모호하게 보이던 회귀를 바로잡았습니다.
- 게시판 관리 화면 곳곳의 역할 없는 빈 컨테이너 중첩을 정리해, 화면 모양은 그대로 둔 채 페이지 구조만 단순해졌습니다.

### Fixed

#### 폼·환경설정·정책 편집

- 관리자 대시보드의 게시판 위젯(최신 게시글·신고 관리 목록)에서 각 항목이 같은 화면 식별자를 공유해 일부 보조 기능·접근성 도구가 항목을 구분하지 못하던 문제를 수정했습니다 — 이제 목록의 각 행이 고유하게 식별됩니다.
- 게시판 본인인증 정책 편집 화면에서 "인증 목적"과 "적용 대상"을 바꿔 저장해도 반영되지 않던 문제를 수정했습니다 — 이제 정책 키·강제 시점·강제 위치를 제외한 모든 항목(인증 목적·적용 대상 등)을 자유롭게 변경할 수 있으며, 변경한 값은 업데이트 후에도 유지됩니다.
- 게시판 생성/수정 폼 "게시글 설정" 탭에서 필드 사이 구분선이 일부 그룹 경계에만 그어지고 일부 필드 사이에는 누락되어 보이던 회귀를 수정했습니다 — 다른 탭과 동일하게 모든 필드 사이에 일관된 구분선이 표시됩니다.
- 게시판 생성/수정 폼에서 탭을 클릭하거나 본문을 스크롤할 때 상단 탭 강조가 따라가지 않던 회귀를 수정했습니다 — 활성 탭이 현재 보고 있는 영역에 맞춰 자동으로 강조됩니다.
- 게시판 환경설정에서 게시판 목록·개별 게시판·게시글 상세의 "SEO 제공 페이지"를 꺼도 검색엔진 사이트맵(sitemap.xml)에 해당 주소가 그대로 남던 문제 수정 — 끈 페이지 유형의 주소는 사이트맵에서 제외됩니다.

#### 삭제·복원·첨부·활동이력

- 답글이 달린 댓글을 삭제하면 그 댓글뿐 아니라 아래 답글까지 함께 사라지던 문제를 수정했습니다 — 삭제된 댓글 자리에는 "삭제된 댓글입니다" 표시가 남고, 다른 사람이 단 답글은 그대로 보입니다.
- 게시판 첨부파일을 내려받을 때 관리자 활동이력에 다운로드 기록이 남지 않던 문제를 수정했습니다 — 이제 첨부파일 다운로드도 업로드·삭제와 동일하게 활동이력에 기록되며, 활동이력에 표시되던 파일 이름이 비어 있던 문제도 함께 해소되어 어떤 파일을 다루었는지 확인할 수 있습니다.
- 로그인한 회원이 첨부파일을 내려받아도 활동이력의 "받은 사람"이 비어 있던 문제를 수정했습니다 — 이제 회원이 첨부를 내려받으면 누가 받았는지 활동이력에 정확히 남습니다. (로그인하지 않은 방문자의 다운로드는 종전과 같이 받은 사람 없이 기록됩니다.)
- 게시판을 삭제하면 소속 게시글·댓글·첨부파일과 첨부 원본 파일까지 함께 완전히 삭제되도록 수정했습니다 — 게시판이 사라진 뒤에도 글·댓글·첨부 데이터와 파일이 남아 용량을 차지하던 문제가 해소됩니다. (신고 접수 내역은 처리 기록 보존을 위해 그대로 유지됩니다.)
- 게시글을 삭제하면 그 글의 댓글·첨부파일도 함께 숨겨지고, 글을 복원하면 함께 부활하도록 수정했습니다 — 사용자가 글 삭제 전에 따로 지운 댓글·첨부는 복원해도 그대로 유지됩니다.
- 삭제된 게시글을 게시판 관리 권한자가 열었을 때 첨부파일이 보이지 않던 문제와, 글과 함께 숨겨진 댓글이 "삭제된 댓글입니다"로 가려지던 문제를 수정했습니다 — 이제 관리 권한자에게는 글과 함께 숨겨진 댓글의 원문과 첨부파일이 그대로 보입니다. (사용자가 직접 지운 댓글은 이전과 같이 가려진 채로 표시됩니다.)

#### 게시판 설정 저장·검증

- 게시판 설정을 여러 게시판에 일괄 적용하던 중 일부 게시판에서 오류가 나면, 어느 게시판에서 실패했는지 알려주고 변경 전체를 자동으로 되돌리도록 수정했습니다 — 일부 게시판에만 설정이 적용되어 게시판마다 설정이 달라지던 문제가 해소됩니다. (전부 적용되거나 전부 적용되지 않으며, 실패 시 안내가 표시됩니다.)
- 첨부 "허용 확장자"를 비워 둔 채 저장하면 안내와 반대로 모든 파일 업로드가 거부되던 문제를 수정했습니다 — 이제 파일 첨부를 사용하는 게시판은 허용 확장자를 최소 1개 이상 입력해야 저장되며, 안내 문구도 이에 맞게 변경되었습니다. (파일 첨부를 사용하지 않는 게시판은 확장자를 비워 둘 수 있습니다.)
- 게시판 샘플 시더가 댓글·답글·첨부 비정규화 카운트를 동기화하지 않아, 시드 직후 게시글 상세 화면에서 댓글/답글 영역이 미표시되던 결함 해소
- 게시판 환경설정 "게시판 설정" 탭의 숫자 입력 필드에 허용 범위 안내(예: 5~100)를 표시하고, 입력칸의 최소·최대값을 실제 저장 한도와 일치시켰습니다. 답변글 깊이는 최대 10단계까지, 첨부 파일 용량 상한도 정상 적용됩니다. 모바일 페이지당 게시글 수 입력칸의 범위 안내와 최소·최대값도 실제 저장 한도를 따르도록 정합화했습니다.
- 환경설정에서 저장한 제목·내용·댓글 글자 수 상한이 실제 게시판 글쓰기 검증과 어긋나 저장 후 글 작성이 막히던 문제 해소 — 제목 최대 1000자, 내용 최대 100000자, 댓글 최대 10000자까지 일관되게 적용됩니다.
- 게시판 설정에서 최소 글자 수를 최대 글자 수보다 크게 저장할 수 있던 문제 해소 — 제목·내용·댓글 각각 최소값이 최대값보다 크면 저장이 막히고, 해당 입력칸에 안내 메시지가 표시됩니다.
- 게시판마다 설정한 "신규 게시글 표시 기간"이 게시글 목록의 [New] 배지에 반영되지 않고 항상 24시간 기준으로 표시되던 문제 해소 — 이제 설정한 기간(예: 1시간, 48시간)에 맞춰 목록의 [New] 배지가 표시됩니다.
- 게시판 이름·설명을 수정한 뒤에도 검색엔진에 이전 제목·설명이 계속 노출되던 문제 해소 — 게시판 정보를 수정하면 해당 게시판 목록 페이지의 검색엔진용 메타 정보가 즉시 갱신됩니다.

#### 복사·권한·역할

- 게시판을 복사할 때 슬러그만 바꿔 저장하면 "관리자 값은 필수입니다" 오류로 저장이 막히던 문제 해소 — 복사 화면에 원본의 관리자·작업 단계 담당자 명단이 미리 채워지고, 권한 구성도 원본과 동일하게 복사됩니다. 복사본에는 새 게시판 전용 관리자 역할이 부여되며, 원본 게시판의 관리자가 복사본 권한에 섞여 들어가던 권한 누수도 차단했습니다.
- 게시판 복사 확인 창의 안내를 정리해, 무엇이 복사되고(설정·권한·관리자) 무엇이 제외되는지(게시글·댓글), 저장 전 새 슬러그를 확인해야 한다는 점을 한눈에 보이도록 했습니다.
- 게시판을 새로 만들거나 복사한 직후 권한 설정 탭의 역할 목록에 게시판 관리자·스텝 역할이 바로 표시되지 않고 새로고침해야 나타나던 문제 해소 — 저장 직후 화면에서도 부여된 역할이 모두 정상 표시됩니다.
- "게시판 관리" 권한을 받은 사용자가 게시판 화면에서는 관리 가능으로 보이는데도 실제로 다른 사람의 글·댓글을 수정하거나 삭제하려 하면 차단되던 문제 해소 — 이제 게시판 관리 권한만으로 타인 글·댓글의 수정/삭제와 수정 화면 진입이 정상 동작합니다.
- 게시판 권한 설정에서 "비밀글 조회" 권한의 역할을 비우고 저장할 때, 안내 메시지에 권한 이름이 영문 키(`posts.read-secret`)로 표시되던 문제 해소 — 다른 권한과 동일하게 "비밀글 조회"로 표시됩니다.

#### 노출 제어·분류·알림·삭제글

- 블라인드 처리된 게시글·댓글의 원문이 권한 없는 사용자에게도 '원글 보기' 버튼으로 그대로 열람되던 문제 해소 — 이제 원문은 게시판 관리자와 작성자 본인만 볼 수 있습니다. 목록의 제목·블라인드 표시와 블라인드 사유 안내는 이전과 동일하게 모두에게 표시됩니다.
- 게시판 목록·검색 결과에서 비밀글의 본문 요약이 권한 없는 사용자에게도 보이던 문제 해소 — 이제 비밀글과 블라인드 글의 본문 요약은 목록·검색에서 표시되지 않습니다. 제목은 이전과 동일하게 표시되며, 본문은 상세 화면에서 열람 권한이 있는 경우에만 볼 수 있습니다.
- 게시판 분류 개수 제한이 적용되지 않아 분류를 무제한으로 추가할 수 있던 문제 해소 — 분류는 최대 50개까지만 추가할 수 있으며, 빈 분류명은 저장되지 않습니다.
- 게시판 분류 입력 안내를 명확히 했습니다 — 분류를 삭제해도 해당 분류의 게시글은 사라지지 않고 "미분류" 필터에서 계속 조회할 수 있음을 안내합니다.
- 게시판에 설정한 금지어가 게시글에만 적용되고 댓글에는 적용되지 않던 문제 해소 — 이제 금지어가 포함된 댓글도 작성·수정 시 동일하게 차단됩니다.
- 분류가 있는 게시판에서 분류를 선택하지 않아도 글이 등록되던 문제 해소 — 이제 분류가 있는 게시판은 글 작성·수정 시 분류 선택이 필수이며, 작성 화면의 분류 항목에 필수 표시와 안내 메시지가 나타납니다.
- 내 알림 설정에서 "게시글 처리 알림"을 꺼도 관리자가 내 글·댓글을 블라인드·삭제·복원할 때 알림이 계속 발송되던 문제 해소 — 이제 이 알림을 끄면 해당 알림을 받지 않으며, 댓글·답글 등 다른 알림과 동일하게 개인 설정이 적용됩니다.
- 관리자 회원 수정 화면에 게시판 알림 설정 영역이 보이지 않던 문제 해소 — 이제 "게시판 알림 설정" 탭에서 게시글 처리·답변·댓글·답글 알림 수신 여부를 회원별로 켜고 끌 수 있습니다. 회원 상세 화면과 동일한 위치·구성으로 표시됩니다.
- 게시판 환경설정의 본인인증 정책 탭에서 출처 필터(전체/코어/모듈/플러그인)를 골라도 결과가 항상 비어 보이던 문제 해소 — 이 탭은 게시판 정책만 보여주는 화면이므로, 동작하지 않던 출처 필터를 정리했습니다. 강제 시점 필터와 검색은 그대로 사용할 수 있으며, 전체 출처의 본인인증 정책은 환경설정의 본인인증 화면에서 관리합니다.
- 블라인드·삭제된 원글에 답글 작성 폼으로 직접 들어가면 작성 폼이 먼저 열리며 원글의 제목·내용이 보이던 문제 해소 — 이제 블라인드·삭제된 원글에는 답글 작성 화면 진입 자체가 차단되어 원글 내용이 노출되지 않습니다.
- 삭제된 게시글·댓글의 원본을 게시판 관리 권한자가 볼 수 없던 문제 해소 — 이제 게시판 관리 권한이 있으면 삭제된 글·댓글의 원본 제목과 내용을 확인할 수 있습니다. 일반 사용자에게는 이전과 동일하게 노출되지 않습니다.
- 삭제된 게시글 상세에 답변 버튼이 그대로 보이던 문제 해소 — 삭제된 글에서는 답변 버튼이 표시되지 않습니다.
- 삭제된 게시글의 첨부파일이 권한 없는 사용자에게도 노출·다운로드되던 문제 해소 — 이제 삭제된 글의 첨부파일은 게시판 관리 권한자만 보거나 내려받을 수 있습니다.

#### 신고 관리·목록 탐색

- 신고 관리에서 영구삭제 처리된 신고 건을 다른 상태(접수·검토·반려·게시중단)로 되돌리려는 요청이 잘못 성공으로 응답되던 문제 해소 — 영구삭제는 최종 상태이므로 다른 상태로의 변경이 거부됩니다. 같은 상태로의 변경 등 허용되지 않는 상태 전환도 함께 차단됩니다.
- 신고 관리 목록(신고현황)에서 검색 필드(전체·게시글제목·게시판명·작성자명·신고자명)에 단어를 입력해 검색해도 결과가 걸러지지 않던 문제 해소 — 이제 검색 종류를 고르고 검색어를 입력하면 해당 조건으로 신고 목록이 정확히 걸러집니다.
- 게시글 목록에서 분류·검색·페이지를 적용한 뒤 글을 열고 "목록" 으로 돌아오면 적용했던 분류·검색·페이지가 풀려 항상 첫 화면으로 돌아오던 문제 해소 — 이제 글 상세로 들어갔다 나와도 보던 목록 상태가 그대로 복원됩니다.
- 분류가 있는 게시판에서 글 상세의 이전/다음 글 버튼이 다른 분류의 글로 넘어가던 문제 해소 — 이제 이전/다음 글은 현재 보고 있는 글과 같은 분류의 글로만 이동합니다. 분류가 없는 글은 이전과 동일하게 전체 글을 기준으로 이동합니다.
- 답글 상세에서 이전/다음 글 버튼이 자기 글타래와 무관한 엉뚱한 원글로 이동하던 문제 해소 — 답글 상세에서는 이전/다음 글 버튼이 표시되지 않으며, 원글로 이동하려면 답글 상단의 원글 링크를 사용합니다.
